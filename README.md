# 公共交通アクセシビリティ分析ツール

## 路線廃止・減便が地域のアクセシビリティに与える影響を定量的に可視化するWebアプリケーション

---

## 概要

少子高齢化やバス運転者不足を背景に、全国で公共交通の路線廃止・減便が進んでいます。本ツールは、GTFSオープンデータを用いて、路線廃止・バス停削除・減便が住民のアクセシビリティに与える影響を、インタラクティブな地図上でシミュレーション・可視化します。

対象エリア：北海道室蘭市（道南バス）
※ GTFSデータを差し替えることで、全国どの地域にも適用可能な設計です。

---

## デモ画面

### 到達圏の可視化
出発地点と時刻を指定すると、制限時間内に到達可能なバス停を色分けして地図上に表示します。バス停をクリックすると、乗り継ぎ経路（何時何分のバスに乗り、どこで乗り換え、何時に到着するか）がポップアップで表示されます。

### 路線廃止シミュレーション
任意の路線を廃止した場合の影響を可視化します。到達不能になるバス停（赤）、到達時間が悪化するバス停（オレンジ）、影響のないバス停（グレー）を色分けし、廃止前後の経路比較と代替路線を確認できます。

### 時間帯別到達圏
5時〜22時の全時間帯の到達圏を一括計算し、スライダーで時間帯を切り替えながら到達圏の変化を確認できます。時間帯別の到達可能バス停数を棒グラフで表示し、公共交通サービス水準の時間変動を定量的に把握できます。

## 機能一覧

### 1. 到達圏の計算・可視化
- 任意の出発バス停を選択可能
- 出発時刻を5時〜22時の範囲で指定可能（時・分単位）
- 制限時間を15分 / 30分 / 45分 / 60分 / 90分から選択可能
- 到達時間に応じた4段階の色分け表示（0-15分 / 15-30分 / 30-45分 / 45-60分）
- バス停クリックで乗り継ぎ経路をポップアップ表示（使用路線名・乗車時刻・所要時間）

### 2. 路線廃止シミュレーション
- 出発地点から直接乗車可能な路線と、乗り換えで利用する路線をグループ分けして表示
- 高速バス（都市間バス）を自動除外
- 廃止対象路線のルートを地図上に赤色で強調表示
- 影響分析の定量表示（到達不能バス停数、到達時間悪化バス停数）
- 悪化バス停クリックで廃止前の使用路線と廃止後の代替路線を比較表示
- 到達不能バス停クリックで廃止前の経路を確認可能

### 3. バス停削除シミュレーション
- 任意のバス停を選択して削除をシミュレーション
- バス路線は削除バス停を通過扱い（路線自体は維持）
- 削除バス停と徒歩エッジで接続されていたバス停間の歩行経路を自動再構成
- 徒歩圏距離を300m（都市部基準）/ 500m（地方部基準）から選択可能
  - 国土交通省「都市構造の評価に関するハンドブック」（2014年）に基づく基準値
- 悪化閾値をユーザーが1分〜15分の範囲で設定可能

### 4. 減便シミュレーション
3つの減便方式を選択可能：
- **特定路線の便数を半分にする**：選択した路線の便を1本おきに間引く
- **特定路線の便を間引く（N本に1本残す）**：2本〜5本に1本を残す比率を指定
- **全路線一律で削減**：10%〜80%の削減率をスライダーで指定

### 5. 時間帯別到達圏アニメーション
- 5時〜22時の全18時間帯の到達圏を一括計算
- スライダーで時間帯を切り替えると、地図上の到達圏が変化
- 時間帯別の到達可能バス停数を棒グラフで表示
- 朝ラッシュ時と昼間のサービス水準の違いを定量的に比較可能

## アルゴリズムの詳細

### 時刻依存ダイクストラ法
通常のダイクストラ法と異なり、エッジのコスト（移動時間）が「何時にそのノードに到着したか」によって変化します。

- 8:15に東室蘭駅に着けば8:20発のバスに乗れる（待ち5分）
- 8:25に着いたら次の9:00発まで待つ必要がある（待ち35分）

この時刻依存性により、出発時刻によって到達圏が大きく変化することを正確にシミュレーションできます。

### ネットワーク構成
- **バスエッジ**: GTFSのstop_times.txtから、同一便内の連続停車をエッジとして生成。出発時刻・到着時刻付き。
- **徒歩エッジ**: 300m以内のバス停間にHaversine距離ベースで生成。歩行速度67m/分（時速約4km）で所要時間を計算。
- **平日ダイヤ**: calendar.txtに基づき、平日運行の便のみを抽出。

### バス停削除の通過扱いロジック
バス停を削除した場合、そのバス停を含む便について、前後のバス停を直接つなぐエッジを再生成します（バスは停車せずに通過する扱い）。同時に、削除バス停と徒歩エッジで接続されていた近隣バス停同士を、新たに徒歩エッジで直接接続します。

重要な設計判断として、バスエッジで接続されていたバス停間には新規徒歩エッジを追加しません。これにより、バス停削除によって意図しない新規接続（異なる路線間の不正な乗り換え）が生まれることを防いでいます。

## 分析から得られた知見

### 1. 時間帯によるサービス水準の格差
室蘭駅前から60分以内に到達可能なバス停数は、朝8時台で117箇所であるのに対し、昼12時台では36箇所に減少します。これは乗り継ぎ待ち時間の増大が原因であり、昼間の公共交通の脆弱性を示しています。

### 2. ハブバス停の重要性
東室蘭駅東口を削除すると、24箇所が到達不能になります。東室蘭駅東口は12路線が発着する交通ハブであり、この結果はハブバス停の定量的な重要度を示しています。

### 3. 路線廃止の影響の非対称性
44路線中、廃止しても影響ゼロの路線が大半である一方、特定の路線（学生便、ろう学校線など）は廃止により15箇所以上が到達不能になります。路線ごとの代替可能性の差異を定量的に把握できます。

### 4. バス停通過による時間短縮効果
バス停を削除すると、通過扱いにより停車時間が削減され、わずかに到達時間が短縮されるケースが確認されました。これは「バス停の適正配置」の検討に資する知見です。

## テスト

297件の自動テストにより、以下を検証済みです。

- **データ整合性**: エッジの移動時間が正、自己ループなし
- **到達圏計算**: 出発地点が結果に含まれる、到達時間が出発時刻以上かつ制限時間以内、経路復元の時間が単調増加
- **路線廃止**: 廃止後の到達可能数が廃止前以下、新規到達なし、到達時間の改善なし
- **バス停削除**: 削除バス停が結果に含まれない
- **減便**: 減便後の到達可能数が減便前以下
- **時刻変化の整合性**: 制限時間を増やすと到達可能数が増加（または同数）
